# Development Rules

## Never

Never rewrite completed modules.

Never redesign UI without approval.

Never remove APIs.

Never change architecture.

Never change database providers.

Never bypass validation.

Never remove RBAC.

Never remove Audit Logging.

Never hardcode values.

Never disable TypeScript.

Never ignore build errors.

Never claim work is complete without verification.

## Always

Reuse components.

Write production code.

Keep modules isolated.

Maintain backward compatibility.

Run build before completion.

Maintain enterprise UI quality.

Update changelog.

Commit frequently.

Use feature branches.
